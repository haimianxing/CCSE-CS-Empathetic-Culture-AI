## 标题（Working Title）

**QD-Synth: Quality–Diversity Optimization for Scalable LLM Data Synthesis with Theoretical and Empirical Analysis of Semantic Collapse**

（备选）**Beyond Self-Refine: MAP-Elites–Style Archives for Non-Collapsing Synthetic Corpora**

---

## 摘要（Abstract）骨架

**问题**：基于 LLM 的迭代合成（self-instruct、evol-instruct、多轮改写）在规模化时普遍出现 **语义塌缩（mode / semantic collapse）**：指令分布变窄、表面改写增多、难例覆盖不足，导致下游微调 **过拟合捷径特征**。

**观察**：将合成仅建模为 **单标量质量最大化**（或对数似然、单裁判打分）**在分布层面不适定**——需要在 **质量** 与 **行为空间覆盖（多样性）** 间显式权衡。

**方法**：提出 **QD-Synth**：将合成过程表述为 **行为描述子（behavior descriptors）** 张成网格上的 **Quality–Diversity（QD）搜索**；用 **MAP-Elites / 改进型档案库（archive）** 维护 **非支配** 样本集合，配合 **轻量质量守门（quality gate）** 与 **多样性压力（diversity pressure）**。合成管线可与具体领域解耦：**Math、Code、Empathetic Dialogue** 三类仅替换 **描述子定义与裁判接口**。

**理论/形式化贡献**（占位，终稿需与导师打磨严谨度）：  
- 给出 **collapse 的操作性定义**（例如：行为网格占用率下降、档案库熵减、近邻语义重复率上升）；  
- 在 **简化假设**（Lipschitz 质量估计、独立采样噪声）下，给出 **单目标贪心合成相对 QD 档案** 的 **覆盖上界/失败模式**（形式为 **命题 + 证明草图** 或 **可检验推论** —— NeurIPS 审稿对「硬定理」挑剔，若定理弱则改 **「Formal Analysis」** 小节并加强实证）。

**实验**：  
- **主结果**：相对 **Self-Instruct / Evol-Instruct / 单目标过滤** 基线，在 **相同 token 预算** 下，QD-Synth 提升 **难例覆盖率** 与 **下游任务增益**；  
- **跨域**：**GSM8K 风格数学**、**HumanEval 风格代码**、**共情/策略对话**（可用 **小规模** CCSE 子集或 **公开英文共情数据** 作第三域，**避免与 EMNLP 主表重复**）；  
- **塌缩诊断**：报告 **网格占用率、档案熵、近邻重复率** 随迭代步变化。

**伦理**：讨论合成数据 **滥用、污染预训练** 风险；建议 **披露合成管线版本** 与 **可复现随机种子**。

## 1. 引言（Introduction）

### 1.1 动机（ML 社区叙事，而非 NLP 应用叙事）

1. 合成数据已成为对齐与指令微调 **主路径之一**，但 **分布健康度** 常被忽视。  
2. 现有工作多报告 **平均质量**，少报告 **支持集覆盖** 与 **塌缩动力学**。  
3. 本文：**把 QD 从进化策略/机器人领域迁移到 LLM 合成**，并给出 **可落地算法 + 可证伪分析 + 跨任务证据**。

### 1.2 贡献（建议 4 条，与 EMNLP 零重复表述）

1. **QD-Synth 框架**：**行为描述子 + MAP-Elites 档案 + 质量门** 的统一接口；适用于 **任意** 可定义行为标签的合成任务。  
2. **Collapse 形式化**：提出 **可计算诊断指标** 与 **（在假设下）单目标合成覆盖不足** 的分析 —— 与「仅展示 case study 的 collapse」区分。  
3. **系统实证**：**Math + Code + Dialogue** 三域 **同协议** 验证 **QD 一致有效**；**不依赖** 中文客服作为主场景。  
4. **与迭代蒸馏链的关系**：讨论 QD-Synth 与 **DPO / 迭代 RLHF** 数据构造的 **正交性**（数据层 vs 对齐层）。

## 3. 预备与符号

- 行为描述子 \(\phi(x) \in [0,1]^d\)（\(d\) 小，如 2–4 维）：可由 **规则 + 小模型 + LLM 裁判** 估计（实现细节附录）。  
- 质量 \(q(x)\)：同一 **裁判接口**（可为 learned reward model 或 LLM rubric）。  
- 档案 \(\mathcal{A}\)：在 **离散网格** \(\mathcal{G}\) 上，每格保留 **精英**（最高 \(q\)）。

---

## 4. 方法：QD-Synth（核心创新）

### 4.1 与单目标合成对照

| 范式 | 更新规则 | 典型失败 |
|------|----------|----------|
| **Greedy-Quality** | 仅保留高 \(q\) | 难行为格 **空**、近邻 **语义重复** |
| **QD-Synth** | 每格保留精英 + 跨格多样性压力 | 需调 **格分辨率** 与 **裁判噪声** |

### 4.2 算法流程（主文给 Algorithm 伪代码）

1. **初始化**：种子集 \(S_0\)（少量人工或公开子集）。  
2. **For** \(t = 1\ldots T\)：  
   - **提议**：LLM 对档案中样本 **变异/组合**（类似 mutation）生成候选 \(x'\)。  
   - **标注**：计算 \(\phi(x')\)、\(q(x')\)。  
   - **档案更新**：MAP-Elites 规则写入 \(\mathcal{A}_t\)。  
3. **输出**：\(\mathcal{A}_T\) 展平为训练集（可 **配额采样** 平衡格）。

### 4.3 **防 collapse 机制**（对应用户「防 collapse」）

- **网格占用率最大化**：显式目标，非事后看 diversity；  
- **精英替换条件**：仅当 **同格** 质量提升 **或** **新格被占领**；  
- **（可选）novelty 辅助**：候选与档案 **嵌入近邻距离** 下限（计算开销附录）。

### 4.4 与「QD 算法原理」共享而不重复 EMNLP

- EMNLP：**若**用 QD 作 **CCSE 过滤**，仅一句「采用 MAP-Elites 风格配额」+ 引用本文 **预印本/arXiv**。  
- NeurIPS：**完整** 伪代码、复杂度、超参敏感性。

## 5. 理论 / 形式化分析（NeurIPS 区分度所在）

### 5.1 Collapse 的操作定义

- **定义 1（网格塌陷）**：\(|\{g \in \mathcal{G} : \exists x, \phi(x)\in g\}| / |\mathcal{G}| \to 0\)。  
- **定义 2（精英同质化）**：档案内 **平均最近邻相似度** \(\to 1\)。

### 5.2 命题方向（占位，需数学顾问审核）

- **命题（示意）**：若候选生成器在 **固定格** 上 **单峰** 且贪心只接受 **全局 \(q\) 提升**，则 **存在** 行为区域 **永不被访问**（覆盖不足）。  
- **推论**：QD 档案在 **每格精英** 约束下，**下界** 覆盖某类行为区域（在 Lipschitz/有界噪声条件下）。

*若定理不易收紧*：改 **「Formal Empirical Analysis」**：**模拟合成 + 受控合成器** 验证命题直觉。

## 6. 实验设计（与 EMNLP 不重叠）

### 6.1 研究问题

- **RQ1**：QD-Synth 是否在 **同预算** 下提升 **下游** 表现 vs 贪心合成？  
- **RQ2**：**塌缩指标** 是否一致改善？  
- **RQ3**：**行为维度 \(d\)** 与 **网格分辨率** 如何缩放？

### 6.2 任务与数据（NeurIPS 主表）

| 域 | 行为描述子示例 | 下游评测 |
|----|----------------|----------|
| **Math** | 难度、推理步数、是否多步 | GSM8K / MATH 子集微调后 accuracy |
| **Code** | 难度、API 数、是否需调试 | HumanEval / MBPP |
| **Dialogue** | 共情强度、策略类型、冲突强度 | **英文** empathetic / strategy 任务 或 **极小** CCSE 子集 **仅作跨域一致性**，**非主叙事** |

**关键**：NeurIPS **主结果不得依赖「中文客服」**；中文客服可作为 **附录一行** 或 **与 EMNLP 联合发布的扩展实验**，避免审稿人认为 **同一工作拆开发表**。

### 6.3 基线

- Self-Instruct 风格；Evol-Instruct 风格；  
- **单目标质量过滤**（同裁判）；  
- **随机子集**（同规模）；  
- **去重 / 聚类采样**（强基线）。

### 6.4 指标

- **下游**：主任务 accuracy / pass@1；  
- **塌缩**：网格占用率、档案熵、Self-BLEU/嵌入相似度；  
- **成本**：LLM 调用次数、token、墙钟时间。

### 6.5 消融

- 去掉 **多样性**（退化为贪心）；  
- 去掉 **格结构**（改为 K-means 簇）；  
- 改变 \(d\) 与网格分辨率。

## 7. 讨论（Discussion）

### 7.1 发现（预设）

- QD-Synth **一致** 降低塌缩指标并提升下游 —— **尤其在难例**；  
- **过细网格** + **噪声裁判** 会导致 **空格震荡** —— 需 **自适应分辨率**（Future Work）。

### 7.2 局限性

- 行为描述子 **未必客观**；  
- 大规模 **万亿 token** 未测；  
- 与 **真实人类数据** 混合的最佳比例未系统探索。

## 9. 核心创新点 vs 「别人没说过」（可辩护表述）
