# 人工评测协议 (Human Evaluation Protocol)

## 目标
完成500条对话的人工评测，达到Kappa≥0.6，满足Strong Accept要求

## 评测样本
- **样本数**: 500条（从522条中随机抽取）
- **抽样方式**: 随机抽样（seed=42）
- **文件**: `data/raw/human_eval_sample_500.json`

## 样本分布
### 领域分布
- 银行: 118条 (23.6%)
- 电商: 128条 (25.6%)
- 医疗: 115条 (23.0%)
- 电信: 101条 (20.2%)
- 其他: 38条 (7.6%)

### 冲突级别分布
- 高: 150条 (30.0%)
- 中: 162条 (32.4%)
- 低: 150条 (30.0%)
- 其他: 38条 (7.6%)

## 评测维度
基于审稿人要求，评测以下4个维度：

### 1. Empathy-Appropriateness (共情恰当性)
- **评分范围**: 1-5分
- **评分标准**:
  - 5分: 共情非常恰当，完美回应用户情绪
  - 4分: 共情恰当，较好回应情绪
  - 3分: 共情基本恰当，有些不足
  - 2分: 共情不足，回应不当
  - 1分: 缺乏共情，回应不当

### 2. Naturalness (自然度)
- **评分范围**: 1-5分
- **评分标准**:
  - 5分: 完全自然，像真人对话
  - 4分: 较自然，少量不自然表达
  - 3分: 基本自然，部分不自然
  - 2分: 不自然，明显机器感
  - 1分: 非常不自然，难以理解

### 3. Policy-Compliance (策略合规性)
- **评分范围**: 1-5分
- **评分标准**:
  - 5分: 完全合规，策略执行完美
  - 4分: 基本合规，策略执行较好
  - 3分: 部分合规，策略执行一般
  - 2分: 不太合规，策略执行差
  - 1分: 完全不合规，策略错误

### 4. Cultural-Fit (文化适配性)
- **评分范围**: 1-5分
- **评分标准**:
  - 5分: 完美适配中文客服文化
  - 4分: 较好适配文化特点
  - 3分: 基本适配文化特点
  - 2分: 文化适配不足
  - 1分: 文化适配失败

## 评测流程

### 阶段1: 标注员招募与培训（1天）
1. **招募3名标注员**
   - 要求: 中文母语，有客服经验优先
   - 背景: 电商、银行、医疗、电信领域

2. **培训（2小时）**
   - 评测维度说明
   - 评分标准示例
   - 试评10条（不计入最终）
   - 讨论与对齐

### 阶段2: 双盲评测（5-7天）
1. **评测方式**: 双盲评测
   - 每条对话由3名标注员独立评分
   - 标注员不知晓其他标注员评分

2. **评测工具**:
   - 使用在线表格或专用标注工具
   - 记录评分时间（用于质量监控）

3. **质量控制**:
   - 每天抽查10%进行质量检查
   - 发现偏差及时纠正

### 阶段3: 一致性检验（1天）
1. **计算Kappa系数**
   - 每个维度分别计算
   - 3名标注员两两计算，取平均
   - 目标: Kappa ≥ 0.6

2. **如果Kappa < 0.6**
   - 分析原因
   - 重新对齐评分标准
   - 重新评测不一致样本

### 阶段4: 结果汇总与分析（1天）
1. **数据汇总**
   - 计算每个维度平均分
   - 计算总体评分
   - 生成统计报告

2. **对比分析**
   - 对比LLM-Judge评分
   - 分析人工评测vs自动评测差异

## 时间计划
- 第1天: 招募与培训
- 第2-6天: 双盲评测（每天100条）
- 第7天: 一致性检验与汇总
- 第8天: 结果分析
